1. Deskripsi Umum Sistem

E-Konsumsi adalah aplikasi manajemen konsumsi kegiatan perusahaan dengan proses approval bertingkat, kontrol role-permission, pencatatan log audit, pengelolaan stok makanan, dan pengelolaan saldo departemen.

Modul utama: Dashboard, Master Data Departemen, Master Data Makanan, Manajemen User/Role/Permission, Manajemen Event, dan Manajemen Saldo.

Aturan penting implementasi:
  • Approve/Reject status Open hanya untuk Manager dengan departemen yang sama dengan event.
  • Terima/Proses & Konfirmasi Kirim hanya untuk user role Karyawan di Departemen Umum (position: karyawan).
  • Approve/Reject status On Process hanya untuk user role Manager di Departemen Umum (position: manager).
  • Konfirmasi Terima (Close by User) hanya untuk pembuat event.

2.2 Daftar Use Case per Role

Role Use Case Batasan
Admin Kelola semua modul dan data sistem. Akses penuh seluruh fitur.
Manager Departemen Buat event, approve/reject event Open, monitor status. Approval Open harus di departemen yang sama.
Manager Dept. Umum Approve/reject event On Process. Harus role Manager, position manager, dan departemen Umum.
Karyawan Pemohon Buat/edit event (Open/Reject), lihat detail, konfirmasi terima. Tidak boleh akses Master Data.
Karyawan Dept. Umum Terima & Proses event, Konfirmasi Kirim (Close by Umum). Harus role Karyawan, position karyawan, dan departemen Umum.

5. Flowchart Proses Bisnis

5.1 Flowchart Approval Event

StatusNamaAksiPelaku
1OpenApprove ke 2 / Reject ke 5Manager Departemen
2Approved VPApprove ke 3Karyawan Dept. Umum
3On ProcessApprove ke 4 / Reject ke 5Manager Dept. Umum
4Approved VP UmumClose ke 6Karyawan Dept. Umum
5RejectEdit lalu kembali OpenPembuat Event
6Close by UmumClose ke 7 (stok makanan & saldo dipotong)Pembuat Event
7Close by UserFinal-
flowchart TD
    A([Open]) --> B{Manager Dept}
    B -->|Approve| C([Approved VP])
    B -->|Reject| R([Reject])

    C --> D{Karyawan Umum}
    D -->|Terima & Proses| E([On Process])

    E --> F{Manager Umum}
    F -->|Approve| G([Approved VP Umum])
    F -->|Reject| R

    G --> H{Karyawan Umum}
    H -->|Konfirmasi Kirim| I([Close by Umum])

    I --> J{Pembuat Event}
    J -->|Konfirmasi Terima| K([Close by User / Final])

    R --> L([Edit Event]) --> A
Saat status berubah ke Close by Umum, sistem melakukan:
  1. Pengurangan stok makanan (food logs tipe stock_out).
  2. Pemotongan saldo departemen sesuai total konsumsi event.
  3. Pencatatan riwayat mutasi pada log_saldos dengan status keluar.
