1. Deskripsi Umum Sistem

Gambaran umum tentang aplikasi E-Konsumsi beserta modul-modul yang ada

E-Konsumsi adalah Sistem Informasi Manajemen Konsumsi Kegiatan/Event yang digunakan untuk mengelola pemesanan makanan pada event-event perusahaan. Sistem ini mencakup proses persetujuan bertingkat (multi-level approval), manajemen stok makanan, pengelolaan saldo departemen, serta pencatatan log untuk audit.

Modul Utama Sistem:
  • Autentikasi — Login, Register, logout menggunakan Laravel Auth
  • Master Data Departemen — CRUD data departemen/divisi perusahaan
  • Master Data Makanan (Food) — CRUD makanan, manajemen harga & stok, log perubahan
  • Manajemen User — CRUD user dengan detail, departemen, role & permission (Spatie)
  • Manajemen Event — Pengajuan event konsumsi dengan peserta & daftar makanan, approval bertingkat (7 status)
  • Manajemen Saldo — Pengelolaan saldo/anggaran per departemen, mutasi masuk/keluar
  • Dashboard Analytics — KPI, grafik statistik event, konsumsi, saldo, dan user

4.3 Flowchart Approval & Status Event

Event memiliki 7 status dengan alur approval bertingkat:

KodeStatusAksi yang TersediaOleh
1 Open Approve → 2, Reject → 5 Manager Departemen
2 Approved VP Approve → 3 Staff Umum
3 On Process Approve → 4, Reject → 5 Manager Dept. Umum
4 Approved VP Umum Close → 6 Staff Umum
5 Reject Edit → kembali ke 1 (Open) Staff Pembuat
6 Close by Umum Close → 7 (Stok makanan dikurangi) Staff Pembuat
7 Close by User — (Final) — 
flowchart TD
    A([Start: Event Status = Open])
    A --> B{Manager Dept:\nApprove / Reject?}

    B -->|Approve| C["Status = Approved VP (2)"]
    B -->|Reject| REJ["Status = Reject (5)"]

    C --> D{Staff Umum:\nTerima & Proses}
    D -->|Approve| E["Status = On Process (3)"]

    E --> F{Manager Dept. Umum:\nApprove / Reject?}
    F -->|Approve| G["Status = Approved VP Umum (4)"]
    F -->|Reject| REJ

    G --> H{Staff Umum:\nKirim Makanan}
    H -->|Close| I["Status = Close by Umum (6)\n🔽 Kurangi Stok Makanan\n📝 Buat Food Log"]

    I --> J{Staff Pembuat:\nKonfirmasi Diterima}
    J -->|Close| K["Status = Close by User (7)\n✅ Event Selesai"]

    REJ --> L{Staff Pembuat:\nEdit & Ajukan Ulang?}
    L -->|Ya| M["Edit Event → Status = Open (1)"]
    M --> B

    K --> Z([End: Event Selesai])

    style Z fill:#10b981,stroke:#059669,color:#fff
    style REJ fill:#ef4444,stroke:#dc2626,color:#fff

Catatan Penting: Saat event berubah ke status "Close by Umum" (Status 6), sistem secara otomatis mengurangi stok makanan (qty_available) pada tabel foods dan mencatat log pada tabel food_logs dengan tipe "out".
